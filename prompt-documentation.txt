ERSONAL PORTFOLIO WEBSITE DOCUMENTATION USING THE DEWEY DECIMAL CLASSIFICATION (DDC) SYSTEM
Project Title

Responsive Personal Portfolio Website

Project Description

The Personal Portfolio Website is a responsive one-page website developed using HTML5 and Vanilla CSS. The website was designed to showcase the developer's personal brand, technical skills, projects, learning journey, and professional contact information while maintaining a modern, clean and responsive user interface.

Unlike traditional web development, this project adopted a Vibe Engineering approach where the website was developed incrementally through prompt engineering. Each section was designed, reviewed, refined, and improved before proceeding to the next section.

The project follows a mobile-first development methodology while utilizing semantic HTML5 elements such as header, nav, main, section, article, and footer to improve accessibility, maintainability, responsiveness, and search engine optimization.

4.0 DESIGN EVOLUTION THROUGH PROMPT ENGINEERING

Throughout the project, the portfolio evolved through continuous feedback and prompt refinement rather than being generated in a single step.

Several significant design improvements were introduced during development.

4.1 Glassmorphism Header

The original header concept evolved into a glassmorphism-inspired navigation after refinement prompts requested a more modern appearance.

4.2 Resume Button Refinement

The navigation button label changed from Resume to Download CV to better communicate its purpose to recruiters.

4.3 Font Awesome Integration

Unicode characters were replaced with Font Awesome icons to improve scalability, consistency, accessibility, and visual quality.

4.4 Editorial Hero Layout

The Hero section was redesigned from a conventional centered layout into an asymmetrical editorial composition to create a more distinctive visual identity.

4.5 Portfolio Section Order

The original website structure was refined from:

Header → Hero → About → Skills → Projects → Contact

to

Header → Hero → Skills → Projects → About → Contact

This decision prioritizes recruiter behavior by presenting technical competencies before personal information.

4.6 Reusable Section Heading Component

A consistent heading component was developed and reused throughout the Skills, Projects, About, and Contact sections to strengthen visual consistency.

4.7 Improved Project Layout

The Projects section underwent multiple refinements to improve heading spacing, card alignment, and visual hierarchy.

The reusable heading component replaced an earlier layout that restricted the heading's available space.

4.8 Sticky Footer Implementation

Instead of using a fixed-position footer, a Flexbox-based sticky footer was implemented to improve accessibility while preventing content overlap.

4.9 Responsive Refinements

Content-driven media queries were adopted instead of relying on framework defaults, ensuring each breakpoint corresponded to genuine layout requirements.
